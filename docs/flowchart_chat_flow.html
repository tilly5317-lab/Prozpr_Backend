How follow-up questions are handled

Follow-ups are a critical path. They are detected at two layers and dispatched to either a no-engine "explain what you said" service (meta) or back to the topic pipeline with a sub-classified action (continuation).

Why two layers
Layer 1 (intent classifier) decides meta vs continuation. Layer 2 (per-handler sub-classifier) decides narrate / recompute / clarify / redirect.
Key signal
active_intent from the last persisted AgentRun in this session — loaded by build_turn_context before classification.
Top-level types
meta — "why did you say X?" / "explain that point". Answered from history; no engine, no fresh data.
continuation — same topic, new substance. Re-enters the topic branch with history as context.
Action modes
AA: narrate · educate · counterfactual_explore · clarify · recompute_full · recompute_with_overrides · redirect
Rebalancing: narrate · recompute · clarify · redirect
Code
app/services/chat_core/turn_context.py · app/services/ai_bridge/follow_up_service.py · app/services/ai_bridge/asset_allocation/chat.py · app/services/ai_bridge/rebalancing/chat.py

Mini examples

Example 1 — "Why did you suggest a Gold ETF for retirement?"

  1. build_turn_context sets active_intent = "asset_allocation".
  2. Intent classifier returns intent=asset_allocation, is_follow_up=true, follow_up_type="meta".
  3. ChatBrain routes to follow_up_service.generate_follow_up_response.
  4. Universal base + asset_allocation clause assembled; Haiku reads conversation history.
  5. Reply is anchored on the prior assistant turn (the one with the gold-ETF recommendation). No engine re-run.

Example 2 — "Now also include 10% gold and re-plan"

  1. active_intent = "asset_allocation"; classifier returns follow_up_type="continuation".
  2. ChatBrain dispatches to the AA chat handler.
  3. Sub-classifier picks recompute_with_overrides with overrides = {gold: 0.10}.
  4. Engine re-runs with the override; new snapshot is persisted.
  5. Reply leads with what changed; old snapshot is no longer the active one.

Example 3 — "Maybe a bit more aggressive?"

  1. Classifier: follow_up_type="continuation", intent asset_allocation.
  2. Sub-classifier: direction signaled but no actionable value → clarify.
  3. Handler returns one short clarifying question (e.g. "By how much — a small tilt or a major shift?").
  4. Engine is NOT re-run.

Example 4 — Rebalancing follow-up "Show me again"

  1. Active intent rebalancing; classifier returns continuation.
  2. Rebalancing sub-classifier picks narrate.
  3. Handler reads last_agent_runs["rebalancing"].output_payload and re-formats the reply. No re-run.
Worked examples

Example A — "How should I invest 10L for retirement in 20 years?"

User asks how to split money across funds.

  1. Frontend POSTs the message + session id to /api/v1/chat/send.
  2. Auth + user loader resolves the JWT and eager-loads the full user graph (profile, goals, portfolio, holdings).
  3. ChatBrain builds a TurnContext (recent messages + last AI module runs).
  4. Intent Classifier (Claude Haiku) returns asset_allocation.
  5. Risk profile is read from the user's effective risk score (computed by the risk_profiling agent earlier).
  6. Asset Allocation pipeline runs steps 1→7: emergency carve-out, short-term, medium-term, long-term, aggregation, guardrails, presentation.
  7. Output is persisted as a snapshot; chat formatter writes a short brief.
  8. Telemetry row written; reply + snapshot id sent back; frontend renders.

Example B — "How is my portfolio doing this month?"

User asks about their existing holdings.

  1. Frontend sends message to /api/v1/chat/send.
  2. Auth + user loader loads user with portfolios → holdings → fund metadata.
  3. Intent Classifier returns portfolio_query.
  4. Portfolio Query agent reads market_commentary_latest.md + the client's holdings (asset class, sub-category, per-fund).
  5. Skill executor loads portfolio_query.md + guardrails.md and sends a single Claude call with prompt caching.
  6. Guardrails decide: in-scope answer or canned redirect.
  7. Telemetry row written; reply sent back; frontend renders.

Example C — "Am I off-balance? Should I rebalance?"

User wants to know what to buy/sell.

  1. Intent classifier returns rebalancing.
  2. Input builder maps current holdings + the latest ideal allocation snapshot into a RebalancingComputeRequest.
  3. Pipeline runs steps 1→6: cap & spill, compare & decide, tax classification, initial trades under STCG cap, loss-offset top-up, presentation.
  4. Engine returns per-fund target / buy / sell amounts plus warnings.
  5. Chat handler turns it into text + a chart payload.
  6. Frontend renders the message and draws the chart.

Example D — "What's happening with Indian inflation?"

User asks a general macro question.

  1. Intent classifier returns general_market_query.
  2. ChatBrain calls Market Commentary with a 120s timeout (skipped on failure).
  3. Market Commentary scrapes indicators, asks Claude to extract a MacroSnapshot, then generates a markdown doc.
  4. General Chat module writes the final reply, optionally grounded in the macro doc.
  5. Telemetry row written; reply sent back; frontend renders.

Tech at a glance (high level)

  • Frontend — React + Vite + Tailwind in Prozpr_Frontend/. Talks to the backend over HTTPS JSON.
  • Backend — FastAPI on Python with async PostgreSQL. Hosts /api/v1/* routers.
  • Auth — JWT bearer + optional X-Family-Member-Id header to act for a family member.
  • AI layer — bundled package AI_Agents/, loaded via sys.path injection. Each agent is self-contained: input → Claude (mostly Haiku) → structured output.
  • AI bridges — one adapter per intent in app/services/ai_bridge/ that maps DB data to agent inputs and formats the reply.
  • Telemetry — every turn writes a flow-summary row (intent, steps, duration). AI module runs also get their own AgentRun rows.
  • Files as contracts — Market Commentary writes market_commentary_latest.md, Portfolio Query reads it. No direct import between AI modules.
  • Prompt caching — used in intent_classifier and portfolio_query so repeated parts of the prompt don't re-bill on every call.
  • Pure-Python pipelines — Asset Allocation and Rebalancing are deterministic (no LLM in the hot path); LLM appears only as an optional "rationale" step.
